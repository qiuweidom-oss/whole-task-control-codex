# Security Policy

Whole Task Control modifies files in the current user's Codex configuration directory. Security reports involving unintended file modification, backup or restore behavior, path handling, or instruction injection are welcome.

Do not post secrets, private configuration files, or other sensitive data in a public issue. Use GitHub private vulnerability reporting when it is enabled. Otherwise, open a public issue containing only a non-sensitive summary and request a private contact channel.

When reporting a problem, include the operating system, shell or PowerShell version, release version, command used, expected behavior, and sanitized error output.
