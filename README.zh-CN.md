# Whole Task Control for Codex

简体中文 | [English](README.md)

让 Codex 在长任务、多轮纠正和上下文压缩后，仍然围绕同一个目标继续工作。

作者：**qiuwei**。

## 为什么做这个 Skill

它来自一个反复出现的真实问题：Codex 的对话变长并经历多次上下文压缩后，容易丢失整件事的脉络，只跟着最后一句话走，重做已经完成的工作，或者生成与此前决定互相冲突的补丁。

Whole Task Control 不会扩大上下文窗口。它为直接服务用户的 Codex 主代理提供一套任务总控方法，让代理从仍然存在的对话、已确认决定、计划、文件和真实进度中恢复整件事，再决定下一步。

## 它解决什么问题

- 用户只纠正一个局部，代理却推翻了整个已确认方案；
- 上下文压缩或任务恢复后，代理开始跑偏或从头重做；
- 一个具体名字、ID、坐标、颜色或数量被误写成通用规则；
- 讨论方案被错误理解为已经授权修改文件；
- 多个子代理各自输出结果，但没有主代理统一验真和收口；
- 多轮修改后只剩一串互相冲突的补丁，需要用户自己拼最终版。

## 核心功能

- **整体任务理解**：维护目标、堵点、已确认决定、约束、真实进度和下一步。
- **新消息分类**：区分补充、纠正、替代、案例、授权、暂停和状态询问。
- **局部纠正保护**：只更新用户明确修改的部分，保留其他有效决定和成果。
- **上下文恢复**：压缩或接班后，只检查存在且与当前任务相关的目标、计划、文件和进度。
- **授权边界**：讨论不等于施工授权，执行范围不会因一句模糊表达自动扩大。
- **案例与规则分离**：把样例用于验收，不轻易写成只服务一个样例的特例。
- **统一最终正本**：多轮修改后重新输出一份完整版本，不让用户手工合并历史补丁。
- **子代理边界**：主代理理解全局并验收，子代理只完成明确限定的局部工作。

## 适用场景

推荐用于：

- 多轮产品、设计或技术讨论；
- 用户会持续纠正局部细节的任务；
- 研究、计划、执行、测试和验收组成的长流程；
- 上下文压缩后的任务恢复；
- 多个子代理协作、由主代理统一收口；
- 经过多次修改的计划、提示词、规范或实施方案。

不建议用于简单翻译、单句改写、事实问答等自包含、低风险、一步完成的请求，也不供被派出的子代理加载。

## Windows 安装

解压 Release 后，在目录中打开 PowerShell。

先预览，不改文件：

```powershell
.\install.ps1 -DryRun
```

确认后安装：

```powershell
.\install.ps1
```

如果系统阻止本地脚本，可以只为当前窗口临时放行：

```powershell
Set-ExecutionPolicy -Scope Process Bypass
```

## macOS / Linux 安装

先预览：

```sh
sh install.sh --dry-run
```

确认后安装：

```sh
sh install.sh
```

如果发现不受本安装器管理的同名 Skill，安装会停止。只有确认需要替换时，才使用 Windows 的 `-Replace` 或 macOS/Linux 的 `--replace`；原版本会先进入备份。

安装完成后，新建一个 Codex 任务或重启 Codex。

## 怎么使用

安装后的轻量全局规则会在任务依赖历史决定、真实进度、纠正、恢复或多个协同步骤时自动加载完整 Skill。

也可以显式调用：

```text
$whole-task-control 请结合本任务此前确认的决定、约束和真实进度，继续完成下一步。
```

上下文压缩后可以这样调用：

```text
$whole-task-control 请从现有计划、文件和进度记录恢复任务，不要重做已完成部分。
```

## 卸载与恢复

Windows：

```powershell
.\uninstall.ps1
.\restore.ps1
```

macOS/Linux：

```sh
sh uninstall.sh
sh restore.sh
```

卸载会移除本工具管理的内容，并尽量保留后来新增的其他规则；恢复会把相关文件退回最近一次操作前的快照，可能覆盖安装后的修改。

## 安全边界

- 安装、卸载和恢复工具只操作当前用户的 Codex 配置目录，不读取或修改项目源代码及 Claude Code；
- Skill运行期间，Codex只有在用户任务和现有权限明确允许时，才会读取或修改相关项目文件；安装本Skill不会扩大任何权限；
- 不联网、不上传数据、不安装第三方依赖；
- 正式修改前先验证规则标记和 Windows 路径长度；
- 默认拒绝覆盖不受管理的同名 Skill；
- 每次真实操作前创建带时间的完整备份；
- 支持预览、失败回滚、卸载和恢复。

## 它不是什么

Whole Task Control 不是更大的上下文窗口、向量数据库、长期记忆库或任务调度器。它解决的是：如何利用仍然可见或已经持久化的信息，避免在长任务中丢失整件事。
